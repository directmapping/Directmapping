1 - Introduction:

 

The SmooKs Direct Mapping is a project to enable users to relate two different structural data sets. To support clear design and optimize performance for hierarchival strcuture, combination of spindown with tree control is used to selectively hide and show information or nodes.

The project was started as subproject of ERPEL project of the Institute of Software Technology and Interactive Systems (ISIS) a Faculty of Computer Science of the Vienna University of Technology.
The SmooKs Direct Mapping project will be supported and implemented under supervision of both the Electronic Commerce Group and Business Informatics Group
mainly Philipp Liegl, Marco Zapletal , as well as others.

2 - Timeline

 

Project Kick-Off was on 11.11.2011

 

The parent project SmooKs Direct Mapping required a user interface to enable user to define relationship between source and destination document. The current Eclipse Plugin for SmooKs was not anymore discontenued by JBoss. The design was not able to support native direct mapping. Due to this reason there was need to create new tool. The project started with top design overview what is to achieve. A web based tool which will be cloud ready had strong influence on the decision not to create plugins into any desktop development tools. A reasearch on proper web based technology to fulfill requirements was commited.

 

Progress meeting 24.11.2011

 

As the technology a strong growing JavaScript Library jQuery - "write less do more" was agreed. The independence of the jDirectMap project will enable to deliver tool to the jQuery community as jQuery plugin. For implementation already existing plugins will be used. The research of these tools was commited.

 

 

Progress meeting 22.12.2011

 

As part of this project a wide research on currently avaiable jQuery tree plug-ins was conducted. Results (full report to be published shortly) shown that there are fairly robust plugins with lots of features to consider:

DynaTree

jsTree

zTree

 

For the jDirectMap map element a jqGrid plugin was choosen.

 

 

Progress meeting 08.02.2012

 

First demo version was presented. The functionality and design present was approved. A discussion around design of functions was conducted. Also the limits of JQuery and technical difficulties presented. Some of the plugins behaive unexpected in multi plugin enviroment. The export of the mapping result as well as the possible import of the source and destination file was outlined.

 

 

Progress meeting 21 - 23.03.2012

 

Project web site go live! Hosted on Google App Engine
First demo version finished with all 3 tree plugins and published on the web site. Project was moved on GitHub see resources page for more details.
